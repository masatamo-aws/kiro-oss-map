## 🔄 ドキュメント更新・管理ルール

### 📝 更新フロー
- **要件変更時**: requirements.md → specifications.md → design.md → logicalarchitecture.md の順で更新
- **機能追加時**: tasks.md → specifications.md → testscenario.md → testresult.md の順で更新
- **リリース時**: testresult.md → CHANGELOG.md → RELEASE-NOTES.md → PROJECT-STATUS.md を更新
- **品質改善時**: PERFECT-QUALITY-ACHIEVEMENT-REPORT → PROJECT-CLEANUP-REPORT を更新

### 📋 ドキュメント形式・規約
- **Markdown形式** (.md) で統一
- **日本語** で記述（技術用語は英語併記）
- **バージョン管理** - 各ドキュメントにバージョン情報を記載
- **更新日時** - 最終更新日を明記
- **絵文字活用** - 視認性向上のため適切な絵文字を使用
- **リンク管理** - 相互参照リンクを適切に設定
